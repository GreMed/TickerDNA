# TickerDNA

**版本：v0.2.0-beta1**

> 历史项目名：Financial Modeling Assumption Agent（用于代码仓库和内部标识迁移参考）

TickerDNA 是一个基于 Streamlit 的财务建模假设工作台。它从公开披露的年报和监管文件中
提取公司业务拆分与历史财务数据，辅助用户搭建收入、毛利、净利润的多情景预测假设，
并导出结构化的 Excel 工作簿。

核心流程：公司搜索 → 历史业务与财务数据 → 假设与驱动因子 → 预测与情景 → 估值与市场对照（示范） → Excel 导出。

> **本工具不是投资建议。** 所有输出均为基于公开披露和模型估算的假设性预测，
> 不构成任何买入、卖出或持有建议。请结合专业判断使用。

## 推荐示范案例

内置两个经核验的示范案例，覆盖美股和港股两种披露口径：

| 示范案例 | 股票代码 | 数据资料截至日期 | 资料来源 |
|----------|----------|------------------|----------|
| Apple Inc. | AAPL | 2025-10-31 | Apple FY2025 Form 10-K |
| 腾讯控股 | 0700.HK | 2026-04-09 | 腾讯 FY2025 年度报告 |

- **Apple (AAPL)**：美股披露通过 SEC EDGAR 官方 API 读取，分部收入来自 10-K 业务分部附注。
  Apple 在 10-K 中未按分部披露毛利率，相关分部毛利率标记为模型估算。
- **腾讯控股 (0700.HK)**：港股披露通过香港交易所披露易定位最新年度报告，分部收入和利润
  来自腾讯 FY2025 年度报告。

> 内置示范案例为静态快照，不等于实时研究结果。页面会明确标注来源类型（实时官方披露、
> 内置官方快照、混合来源、模型估算）。

## 启动方式

```bash
cd TickerDNA-v0.2.0-beta1
pip install -r requirements.txt
streamlit run app.py
```

启动后在浏览器中打开应用页面，按页面引导依次完成：搜索公司 → 读取公司资料 →
查看历史业务拆分 → 调整假设 → 查看预测 → 查看估值对照（示范） → 导出 Excel。

### 环境要求

- Python 3.9 或更高版本
- 依赖见 `requirements.txt`

### 配置外部数据源（可选）

不配置 OpenAI API Key 时，公司搜索和公开披露自动拆分仍可使用。只有在结构化主营构成
和年报解析都无法通过校验时，应用才会生成明确标记的占位拆分。

如需配置，复制 `.env.example` 为 `.env` 并填写：

- `OPENAI_API_KEY`：用于公司资料研究与结构化生成
- `SEC_USER_AGENT`：美股 SEC EDGAR 官方 API 要求的联系邮箱（格式 `TickerDNA your-email@example.com`）

也可在应用侧边栏的「数据源配置」中临时填写，仅在当前会话生效。

## 当前支持范围

- **公司搜索**：支持中、美、港股上市公司搜索（A 股代码末尾加 `.SH`/`.SZ`，港股加 `.HK`，美股用英文代码）
- **历史业务与财务数据**：自动读取官方年报和监管文件，按产品/地区/行业生成有来源的业务拆分，区分公司披露分部与模型估算分部
- **假设与驱动因子**：逐年度修改 Base 收入增长率和毛利率，设置 Bull / Bear 统一振幅，经营费用率和其他损益率采用各情景共用的逐年假设
- **预测与情景**：自动输出 Bull / Base / Bear 三情景收入、毛利、净利润预测，支持图表切换
- **估值与市场对照（示范）**：仅 Apple 和腾讯两个内置案例提供静态估值演示，将 Base 预测与模拟一致预期、参考市值进行对照。尚未接入真实券商一致预期和实时行情；普通公司仍不提供真实估值与券商预期功能
- **Excel 导出**：导出带格式、图表和明细的 Excel 工作簿，预测 Sheet 使用公式计算（修改假设值后自动更新）

## 已知限制

- **估值与市场对照为功能示范**：当前仅 Apple 和腾讯案例提供静态估值演示；尚未接入真实券商一致预期和实时行情；普通公司仍不提供真实估值与券商预期功能；不构成投资建议
- **不提供 EPS、现金流、三张表、券商预期或价格预测**：MVP 范围仅覆盖收入、毛利、净利润预测和 Excel 导出，不构建资产负债表和现金流量表
- **不接入公告归因、资金流、市场传闻**：不包含融资、营运资本、折旧摊销等完整三表逻辑
- **分部毛利率在部分公司未在 10-K 中披露**：如 Apple 未按分部披露毛利率，相关数值标记为模型估算，不会伪装为公司披露
- **内置示范案例为静态快照，不等于实时研究结果**：部分公司使用经核验的内置官方快照，并非实时抓取的最新数据，页面会明确标注来源类型
- 扫描版 PDF 或极复杂跨页表格可能无法自动解析，此时会保留官方链接并提示

## Excel 工作表结构

导出的 Excel 工作簿包含 **7 个工作表**，固定顺序如下：

| 序号 | 工作表名称 | 说明 |
|------|------------|------|
| 1 | 基期实际 | 各业务分部基期收入、毛利率、利润等实际值 |
| 2 | 假设 | Base 情景逐年度收入增长率和毛利率，Bull / Bear 振幅，经营费用率和其他损益率 |
| 3 | 假设依据 | 每个 Base 假设的来源、预测方法、置信度和用户修改状态 |
| 4 | 预测汇总 | Bull / Base / Bear 三情景收入、毛利、净利润汇总 |
| 5 | Bull | Bull 情景逐年度逐分部预测明细（公式计算） |
| 6 | Base | Base 情景逐年度逐分部预测明细（公式计算） |
| 7 | Bear | Bear 情景逐年度逐分部预测明细（公式计算） |

> Bull / Base / Bear Sheet 中的预测值是公式计算，修改「假设」Sheet 中的数值后自动更新。

## 模型逻辑

- **收入**：各业务分部基期收入 × 逐年度 Base 增长率，逐年复合增长
- **业务拆分**：可选择按产品、按地区、按行业或自定义口径；搜索公司后会按已知披露情况预选默认口径，若财报未披露用户指定口径，会先提示确认，坚持后才生成明确标记的估算拆分
- **分部披露指标**：除收入和毛利率外，会尽量保留年报中可取得的分部净利润、营业利润、税前利润或 EBITDA，并自动计算对应分部利润率
- **毛利**：各分部收入 × 逐年度 Base 毛利率后加总
- **情景**：Bull = Base + 统一振幅；Bear = Base - 统一振幅
- **预测首年**：采用最新基期年度 + 1，如最新基期为 FY2025，则第一个预测年度为 FY2026E
- **经营费用率与其他损益率**：不区分 Bull / Base / Bear；逐年单独可编辑，也可设置相对基期的年度线性变动，默认每年变动为 0
- **税前利润**：毛利 - 经营费用 + 其他损益
- **净利润**：税前利润 - 所得税；亏损年份不确认税收收益
- **单位**：默认人民币百万元

## 数据来源

应用中的数据有几种不同的来源，页面上会明确标注，不会混淆：

| 来源类型 | 说明 | 怎么识别 |
|----------|------|----------|
| 实时官方披露抓取 | 从交易所或监管网站实时读取的最新数据 | 页面显示「实时官方披露」 |
| 内置官方快照 | 已经内置在程序中的经核验的历史官方数据 | 页面显示「内置官方快照」 |
| 混合来源 | 一部分来自官方披露，一部分来自内置快照 | 页面显示「混合来源」 |
| 模型估算 | 当官方数据不可用时，模型生成的估算值 | 页面显示「模型估算」并有 ⚠ 警告 |

**重要原则：**
- 内置快照和模型估算永远不会伪装成实时公司披露
- 没有来源时会明确提示，不会凭空生成数字
- 基期实际值、估算值、用户修改值、预测值会分别标注

### 数据源通道

- **美股**：SEC EDGAR 官方 API（`submissions`、`companyfacts`、10-K / 20-F 原文），分部收入通过 Inline XBRL 维度和年报正文提取
- **港股**：香港交易所披露易匹配发行人并定位最新年度报告
- **A 股**：先按证券代码读取结构化主营构成，缺失时自动从巨潮资讯或交易所定位年度报告 PDF 并解析
- 所有远程公司目录和披露 JSON 都会缓存在 `.cache/company_data/`，公司目录默认缓存 24 小时，披露查询默认缓存 6 小时
